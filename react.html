Что такое React?

React — это библиотека для постороения гибких пользовательских интерфейсов

XHP

  • PHP-шаблонизатор с XML-подобным ситаксисом
  • Разработан Facebook в 2010 году

Пример кода на XHP

			$html = <ui:layout>
				<div id="somediv">
					<p>Hello World!</p>
				</div>
			</ui:layout>;
			echo $html;

Virtual DOM

Виртуальная легковестная копия реального DOM

JSX

  • XML-подобный синтаксис для описания шаблонов React
  • Разработан японской компаний DeNa в качестве эксперимента
  • Основная цель — легкость и простота описания шаблонов

Пример JSX

Чистый React

			React.createElement(Layout,
				{ fixed: true, className: 'ui-layout' },
				React.createElement('div', { id: 'somediv' }, 'Not cool')
			);

React + JSX

			<Layout className="ui-layout" fixed>
				<div id="somediv">JSX rulezz!</div>
			</Layout>

Архитектура

  • React — это библиотека, а не framework!
  • React — это View из MVC/MVVM/MVP
  • Используй где угодно (хоть с Backbone), но не забывай об особенностях

Во frontend нет «best practice» архитектуры

Flux

Термин, придуманный для обозначения однонаправленного потока данных

Схема Flux

Redux

  • Библиотека для управления состоянием приложения
  • Разработчик: Даниил Абрамов
  • Расширяет Flux
  • Полагается на «иммутабельность»

Схема Redux

MobX

  • Библиотека для управления состоянием приложения
  • Разработчик: Michel Weststrate
  • Собственный механизм управлению состоянием на основе Observable
  • «Мутабельное» состояние

Внедрение и применяемость

  • Нет готовой архитектуры
  • Необхоима среда для компиляции
  • Большие затраты на первоначальное внедрение
  • Хорошо подходит для больших, сложных SPA приложений

Интеграция

  • Легко интегрируется
  • Не любит делиться управлением DOM
  • Может «подружиться» с другими библиотками изменяющими DOM

Итоги

Плюсы

  • Модульность и композиция
  • Virtual DOM
  • JSX

Минусы

  • Необходимость компиляции
  • Ориентированность на большие приложения
  • Недостаточное количество готовых модулей
